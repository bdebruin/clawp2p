Node setup

Install a ClawP2P node

Run this on any machine you want to contribute to the network. An AI agent can walk you through every step — paste these instructions or share this URL.

For AI agents: this page is structured for you to follow step by step. Machine-readable metadata is at /llm.json. Each step has a command block you can give the user to run. The verify section at the bottom tells you what a working node looks like.

Prerequisites

  • ✓Python3.10 or newer
  • ✓Dockerany recent version
  • ✓Gitto clone the repo
  • ✓pipcomes with Python

Steps

01

Clone the repo

git clone https://github.com/bdebruin/clawp2p.git cd clawp2p
02

Install dependencies

All dependencies are pure Python. cryptography handles Ed25519 signing. flask runs the node API. requests handles bundle transfers.

pip install -r requirements.txt
03

Generate your node keypair

Each node has an Ed25519 identity. Agents use your public key to verify that bundles you forward haven't been tampered with.

python3 -c " from signing import generate_keypair, save_private_key from pathlib import Path Path('keys').mkdir(exist_ok=True) kp = generate_keypair() save_private_key(kp, Path('keys/node.pem')) print('Public key:', kp.public_id) "

Save the printed public key — you'll add it to your trusted_keys.txt if you want to run test agents signed by yourself.

04

Add trusted signing keys

Your node only runs agents signed by keys you've explicitly trusted. Create trusted_keys.txt with one ed25519:<hex> key per line.

# Add your own public key (from step 03) to trust your own agents echo "ed25519:YOUR_PUBLIC_KEY_HEX" > trusted_keys.txt

Blank lines and lines starting with # are ignored. There is no trust-on-first-use — every key is an explicit decision.

05

Pull the agent base image

Agents run inside a Docker container. The base image is a minimal Python 3.11 environment with no extra capabilities.

docker pull clawp2p/agent-base:0.1

If the image isn't yet published, build it locally from the repo:

docker build -t clawp2p/agent-base:0.1 docker/
06

Start the node

The node daemon listens for incoming bundles on port 7777 by default. Set CLAWP2P_PORT to change it.

python3 node.py

Or with environment overrides:

CLAWP2P_PORT=7777 CLAWP2P_NODE_ID=my-node python3 node.py
07

Run the test suite

Verifies signing, tamper detection, path traversal rejection, and the full round-trip. All 29 tests should pass before you accept any external bundles.

python3 -m pytest test_bundle.py -v

How to verify a working node

✓ Node responds at http://localhost:7777/status — returns JSON with node_id, uptime, and public key
✓ http://localhost:7777/policy — returns your configured resource ceilings and egress allowlist
✓ All 29 pytest tests pass — if any fail, the rename or install was incomplete
✓ POST http://localhost:7777/bundle with a valid .claw file returns 202 Accepted
